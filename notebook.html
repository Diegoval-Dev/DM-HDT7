Paso 1: Crear variables dicotmicas para las categoras de precio

Para aplicar modelos de clasificacin binaria como la regresin logstica, es necesario transformar la variable categrica que representa el precio de la vivienda en variables dicotmicas. En ejercicios anteriores se clasificaron las viviendas en tres categoras: cara, media y econmica.

En este paso, crearemos tres nuevas columnas binarias:

  • cara_o_no: toma valor 1 si la vivienda es cara, 0 en caso contrario.
  • media_o_no: toma valor 1 si la vivienda es de precio medio, 0 en caso contrario.
  • economica_o_no: toma valor 1 si la vivienda es econmica, 0 en caso contrario.

Estas variables permitirn construir modelos independientes para predecir cada categora.

Anlisis del Paso 1: Variables dicotmicas

Las nuevas columnas cara_o_no, media_o_no y economica_o_no fueron creadas exitosamente a partir de la variable categrica PriceCategory. Cada una representa una clase como una variable binaria (1 si pertenece a esa clase, 0 si no).

Este paso es esencial para aplicar modelos de clasificacin binaria como la regresin logstica. Al usar estas variables, se podr entrenar un modelo independiente para predecir si una vivienda pertenece o no a cada una de las categoras definidas.

El resultado obtenido muestra que las variables se asignaron correctamente. Por ejemplo, una vivienda con PriceCategory = "Cara" tiene cara_o_no = 1 y media_o_no y economica_o_no iguales a 0.

Paso 4: Anlisis del modelo (significancia, multicolinealidad, correlaciones)

Antes de considerar el modelo de regresin logstica como confiable, es importante evaluar:

  1. Significancia estadstica de los coeficientes, usando p-valores. Esto nos dir si las variables realmente contribuyen a explicar la probabilidad de que una vivienda sea Cara.

  2. Multicolinealidad, evaluada con el Factor de Inflacin de la Varianza (VIF), para detectar si algunas variables explicativas estn altamente correlacionadas entre s.

  3. Correlacin entre predictores, a travs de una matriz de correlacin, lo cual tambin puede evidenciar redundancia entre variables.

Este paso tiene como objetivo evaluar la estabilidad, interpretabilidad y fiabilidad del modelo.

Paso 6: Verificacin de sobreajuste

Para evaluar si el modelo de regresin logstica est sobreajustado a los datos de entrenamiento, se comparan las mtricas de desempeo entre los conjuntos de entrenamiento y prueba.

Adems, se genera una curva de aprendizaje que muestra cmo evolucionan el accuracy de entrenamiento y validacin a medida que se incrementa el tamao del set de entrenamiento.

Esto permite identificar si el modelo mejora de forma estable con ms datos, o si hay una brecha significativa entre el error de entrenamiento y el de validacin.

In[28]:
from sklearn.model_selection import learning_curve
import numpy as np
import matplotlib.pyplot as plt

# 1. Accuracy en el conjunto de entrenamiento
y_train_pred = model.predict(X_train)
train_acc = accuracy_score(y_train, y_train_pred)
print(f"Accuracy en entrenamiento: {train_acc:.4f}")

# 2. Generar curva de aprendizaje
train_sizes, train_scores, test_scores = learning_curve(
    LogisticRegression(max_iter=1000, solver='liblinear'),
    X_train, y_train,
    cv=5,
    scoring='accuracy',
    train_sizes=np.linspace(0.1, 1.0, 10),
    n_jobs=-1,
    shuffle=True,
    random_state=42
)

# Promedio y desviacin estndar
train_scores_mean = np.mean(train_scores, axis=1)
test_scores_mean = np.mean(test_scores, axis=1)

# 3. Graficar curva de aprendizaje
plt.figure(figsize=(8,6))
plt.plot(train_sizes, train_scores_mean, label='Accuracy en entrenamiento')
plt.plot(train_sizes, test_scores_mean, label='Accuracy en validacin')
plt.xlabel('Tamao del conjunto de entrenamiento')
plt.ylabel('Accuracy')
plt.title('Curva de aprendizaje - Regresin logstica')
plt.legend()
plt.grid(True)

Este comportamiento es esperable cuando:

  • Se tienen muchas variables predictoras.
  • El modelo es suficientemente flexible como para aprender ruido.
  • Las clases estn balanceadas, pero se quiere maximizar exactitud.

Conclusin:

Aunque el modelo presenta un leve sobreajuste (perfecto en train, pero <1.0 en test), su rendimiento en validacin sigue siendo muy alto. Para reducir an ms el sobreajuste, podran aplicarse estrategias como:

  • Regularizacin (L1 o L2).
  • Seleccin de variables significativas.
  • Reduccin de dimensionalidad.

Sin embargo, no hay seales crticas que afecten gravemente la generalizacin, por lo que este modelo sigue siendo una base slida para comparaciones futuras.
